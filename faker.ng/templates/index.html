Faker NG

Faker.ng is a read-only web service for generating Nigerian fake data such as names, addresses, and phone numbers.

About View Docs

Introduction

In January 2016, AbujaDevMeetup members set out to create a data store for Nigeria's local information which can be used anytime sample data is needed as against the most available ones whose data are usually samples from the west.

Faker NG is a simple API that allows you pull sample names, email addresses, phone numbers, etc to populate your database during development

It is a simple developer tool that helps you to generate sample data for your application. It is inspired by different Faker packages around. This is specifically aimed and targetted at Nigerian names, email addresses, cities, streets, and general information that will help you speed up and imitate a real life information used in your application.

It's built in Python with Flask using Parse as a data store and returns data in a JSON format e.g. 

[{
                         "surname": "Musa", 
                         "email": "musa.dinshiya@gmail.com", 
                         "firstName": "Dinshiya"
                    }, 
                    {    "surname": "Essien", 
                         "email": "essien.abdul@yahoo.com", 
                         "firstName": "Abdul"
                    }, 
                    {    "surname": "Adedeji", 
                          "email": "adedeji.musa@yahoo.com", 
                          "firstName": "Musa"
                    }]

Versioning

The API is based at http://faker.abujadevmeetup.com/api/v1/faker

It is being continually developed. Changes might not have backward compatibility. You can always access the API using the version number. e.g. http://faker.abujadevmeetup.com/api/~version-name/faker

Entity and Properties

In current Iteration (v1), there is only one entity presently. which is People

The properties are:

  • FirstName - First Name
  • surname - Surname
  • Email - Email Address
We look forward to adding others such as State of Origin, Tribe, etc

List of Endpoints

Currently, there are 2 end points:

Endpoint URI
People http://faker.abujadevmeetup.com/api/v1/faker/people/
Email http://faker.abujadevmeetup.com/api/v1/faker/email/

Examples

Few Examples of Use case is stated below. Just GET the URI passed, a JSON data is returned to your application.

Getting People

To get X number of People,
Endpoint people
Parameter(s) Int: numberOfPeople (3)
URI http://faker.abujadevmeetup.com/api/v1/faker/people/3
Result 
[{"surname": "Musa", 
                     "email": "musa.dinshiya@gmail.com", 
                     "firstName": "Dinshiya"
                     },
                     {"surname": "Essien", 
                     "email": "essien.abdul@yahoo.com", 
                     "firstName": "Abdul"
                     }, 
                     {"surname": "Adedeji", 
                     "email": "adedeji.musa@yahoo.com", 
                     "firstName": "Musa"
                     }]

Getting Only Emails


Endpoint email
Parameter(s) Int: numberOfEmail (3)
URI http://faker.abujadevmeetup.com/api/v1/faker/email/3
Result 
[{"email": "musa.dinshiya@gmail.com"}, 
                    {"email": "essien.abdul@yahoo.com"}, 
                    {"email": "adedeji.musa@yahoo.com"}
                    ]
